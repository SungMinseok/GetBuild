# QuickBuild 변경 이력

## [v2.0.0] - 2025-10-26

### UI 전면 개편
- 메인 UI를 스케줄 리스트 중심으로 완전히 재설계
- 카드형 스케줄 아이템 위젯으로 시각적 개선
- 상단 헤더에 스케줄 관리 버튼 배치 (새 스케줄, 새로고침)
- 하단에 실시간 로그 영역 추가 (스플리터로 크기 조절 가능)

### 스케줄 관리 기능 강화
- 스케줄 생성/편집 다이얼로그 추가 (상세 설정 가능)
- 반복 설정 추가: 일회성(Once), 매일(Daily), 주간(Weekly)
- 주간 반복 시 특정 요일 선택 가능 (월~일)
- 스케줄 활성화/비활성화 토글 기능
- 스케줄 수동 실행 버튼 추가 (각 카드에 ▶ 버튼)
- 스케줄 편집 기능 추가 (기존 스케줄 수정 가능)
- 스케줄 삭제 시 확인 다이얼로그 표시

### Core 모듈 확장
- ScheduleManager에 daily/weekly 반복 기능 추가
- UUID 기반 스케줄 ID 관리 시스템 구현
- 스케줄 CRUD 메서드 완전 구현 (add/update/delete/toggle)
- get_due_schedules 메서드에 반복 타입 처리 로직 추가

### QThread 백그라운드 실행
- WorkerThread 클래스 추가 (범용 백그라운드 작업)
- ScheduleWorkerThread 클래스 추가 (스케줄 전용)
- 시그널 기반 상태 관리 (progress, status, finished, log)
- 여러 스케줄 동시 실행 가능
- UI 블록킹 없이 작업 실행

### UI 컴포넌트 모듈화
- ui/schedule_dialog.py 추가 (스케줄 생성/편집 다이얼로그)
- ui/schedule_item_widget.py 추가 (스케줄 카드 위젯)
- ui/__init__.py 추가 (UI 모듈 패키지화)

### 유틸리티
- migrate_schedule.py 추가 (기존 schedule.json 마이그레이션)
- 자동 백업 생성 기능
- 샘플 스케줄 생성 기능

### 문서화
- README_V2_UI.md 추가 (새 UI 완전 가이드)
- README_REFACTORING.md 업데이트 (최신 변경사항 반영)
- SCHEDULE_TEST_GUIDE.md 작성 (스케줄 테스트 가이드)

### 기타
- index_v2.py 추가 (새 메인 UI 파일)
- index.py 보존 (기존 UI 백업)
- 테스트(로그) 실행 옵션 추가
- 실행 로그 실시간 표시 개선

## [v2.0.1] - 2025-10-26 (추가 개선)

### UI 개선
- 헤더 높이 축소 (80px → 40px)
- 버튼 크기 조정 (더 컴팩트하게)
- 제목 폰트 크기 축소 (18pt → 12pt)
- 여백 최적화
- 스타일시트 자동 로드 비활성화 (성능 개선)

### 스케줄 관리 기능 강화
- 스케줄 복사 기능 추가 (ScheduleManager.copy_schedule)
- 스케줄 아이템에 복사 버튼 추가 (📋 복사, 보라색)
- JSON 파싱 오류 시에도 강제 삭제 가능하도록 개선
- 강제 삭제 시 자동 백업 생성 (schedule.json.error_backup)
- 삭제 오류 발생 시 사용자에게 강제 삭제 옵션 제공

### 표시 개선
- 스케줄 아이템에 AWS URL 마지막 부분만 표시
  * 예: https://.../sel-game5 → "sel-game5"
  * URL 없을 경우: "없음"
- 스케줄 아이템에 Branch 정보 명시적으로 표시
  * 포맷: "🌿 Branch: game_dev" 또는 "🌿 Branch: 없음"
- 스케줄 카드 정보 라인 개선
  * "⚙️ 옵션 | 🌐 AWS: xxx | 🌿 Branch: xxx"
- 버튼 간격 축소 (5px → 3px)

### 로그 관리
- 로그를 log/ 폴더에 날짜별로 자동 저장
- 로그 파일명 형식: log_YYYYMMDD.txt (예: log_20251026.txt)
- 기존 로그에 이어쓰기 (날짜별 누적)
- 로그 디렉토리 자동 생성
- 파일 저장 오류 시 콘솔에만 출력 (UI 영향 없음)

### Core 모듈 개선
- ScheduleManager.delete_schedule에 force 파라미터 추가
- ScheduleManager.copy_schedule 메서드 추가
- UUID 기반 고유 ID 자동 생성
- 복사 시 이름에 "(복사본)" 자동 추가

### 버그 수정
- schedule.json 파싱 오류 시 프로그램 크래시 방지
- 삭제 실패 시 적절한 에러 메시지 표시
- 로그 파일 인코딩 UTF-8로 명시

### 문서
- CHANGELOG.txt 생성 및 모든 변경사항 기록
- 날짜별 변경 이력 관리
