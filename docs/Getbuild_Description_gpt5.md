## QuickBuild v2: 스케줄 기반 빌드/배포 자동화 도구

### 개요
- **목적**: QA·개발 조직에서 반복적인 빌드 복사, 서버 패치, 빌드 트리거 업무를 스케줄로 자동화해 인적 오류를 줄이고 처리 속도를 향상.
- **형태**: Windows용 데스크톱 앱(PyQt5), 스케줄 중심 UI.
- **핵심 가치**: 정해진 시각/주기에 맞춘 무인 실행, 진행 상태 가시화, 실행 로그 축적.

### 도입 배경
- **수작업 빌드 운영의 한계**: 야간/새벽 빌드 반영 지연, 네트워크 드라이브 대용량 복사 실패 재시도의 번거로움, 서버 패치 시 수동 절차의 오류 가능성.
- **QA 일정 압박**: 테스트 시작 시간 준수 필요, 환경 준비(클라/서버) 병목 발생.
- **해결 방향**: 스케줄러와 일관된 작업 옵션으로 “클릭 없이도” 환경이 준비되도록 자동화.
- **AI 개발도구 활용**: UI 스캐폴딩, 스레드/시그널 템플릿, 예외 흐름, 메트릭 설계 템플릿을 단기간에 생성·보완하여 리드타임 단축.

### 주요 기능
- **스케줄 관리**: 생성/편집/복사/활성화 토글/수동 실행/중지. `schedule.json` 기반 관리, 시간·반복(요일) 지원.
- **실행 옵션(작업 단위)**:
  - **클라복사**: 최신 빌드의 `WindowsClient`만 선택 복사.
  - **서버복사**: 최신 빌드의 `WindowsServer`만 선택 복사.
  - **전체복사**: 빌드 전체 폴더 트리 복사.
  - **서버패치**: AWS 콘솔(내부 `AWSManager`)을 통한 컨테이너 업데이트.
  - **빌드굽기**: TeamCity 빌드 트리거.
  - 그 외 서버업로드/업로드+패치(좌측 TODO 표시: 추후 구현).
- **최신 빌드 자동 판별**: 소스 경로에서 빌드명 포함 디렉토리를 수정시각 기준으로 최신 선택.
- **대용량 복사 로직**: 경로 검증, 목적지 자동 생성, 읽기 전용 속성 해제, 재시도(기본 1회), 실패 파일 목록 집계.
- **실시간 상태/로그**: 실행 중 개수 요약, 작업별 상태 표시, 화면/파일 동시 로깅(`log/log_YYYYMMDD.txt`).
- **자동 업데이트**: `version.json` 확인, 다운로드 진행률 UI, 무중단 알림.
- **설정/구성**: `config.json`, `settings.json`에서 경로·빌드명 프리셋, `UI` 모듈로 스케줄 입력 다이얼로그 제공.

### 아키텍처 개요
- **UI 레이어**: `QuickBuildApp`(메인 윈도우), `ScheduleDialog`, `ScheduleItemWidget`.
- **코어 레이어**: `ConfigManager`, `ScheduleManager`, `BuildOperations`, `ScheduleWorkerThread`.
- **인프라**: `AWSManager`(서버 패치/TeamCity), `AutoUpdater`(선택적).
- **데이터**: 설정/스케줄 JSON, 일자별 텍스트 로그.

### 동작 흐름
1. 스케줄 등록(시간/반복/옵션/경로/브랜치 등).
2. 1초 단위 타이머가 현재 시각과 스케줄 매칭 → 실행 대상 도출.
3. 옵션별 작업 실행(별도 스레드), 진행 상태 UI 업데이트.
4. 완료/오류 이벤트 처리 → 요약/로그 반영.
5. 로그 파일로 누적되어 사후 분석·시각화의 데이터 소스로 활용.

## 실제 효과 측정과 시각화(권장 KPI)
아래 항목은 동일 기준으로 타 앱과도 비교·통합이 용이하도록 설계했습니다.

- • **환경 준비 리드타임(TTEnv)**: 스케줄 시작→완료까지 평균 소요 시간.
- • **스케줄 성공률**: 성공/전체 실행 비율(일·주·월).
- • **복사 처리량**: 파일 수·데이터량 대비 처리 시간(MB/s, files/min).
- • **재시도/실패율**: 실패 파일 수 비율, 사유 분류(권한/경로/점유 등).
- • **야간 무인 실행률**: 20:00~08:00 실행 건수/전체.
- • **수동 개입 감소**: 운영자가 개입한 이벤트 수(중지/재시작/수정) 추세.
- • **빌드 가용 시각 안정성**: 규정 시각 대비 준비 완료 편차의 표준편차.

### 데이터 수집 방법
- **로그 파싱**: `log/log_YYYYMMDD.txt`에서
  - 시작/완료 로그 타임스탬프 추출 → TTEnv.
  - “files copied”, “skipped” 문구 → 처리량·실패율.
  - “✅ 완료/❌ 실패” → 성공률.
- **보강 권장**: 각 작업 시작·종료를 키-값 JSON 한 줄 로그로 병행 출력하면 파이프라인이 견고해집니다.
  - 예: `{"ts":"2025-10-29T01:23:45","scheduleId":"...","option":"전체복사","event":"end","files":1234,"skipped":5,"durationSec":842}`

### 시각화 예시(그래프 유형)
- **라인 차트**: 일자별 TTEnv 평균/중앙값 추세.
- **히스토그램/박스플롯**: TTEnv 분포(피크타임·야간 비교).
- **스택 막대**: 성공/실패 및 실패 사유 구성비.
- **콤보 차트**: 파일 수 vs 처리 시간, 처리량 추세.
- **레이더 차트**: 앱 간 KPI(성공률, TTEnv, 실패율 등) 종합 비교.
